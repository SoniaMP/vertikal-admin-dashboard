# AUTH — Autenticación y roles

Login interno (admin e instructor) vía NextAuth con credenciales, middleware que protege rutas según rol, navegación contextualizada, guardas de propiedad sobre cursos y participantes para instructores. Incluye normalización y verificación previa de DNI en flujos públicos.

## Reglas

- Las contraseñas se almacenan hasheadas; nunca se guardan ni transmiten en texto plano.
- El middleware de Next.js (`src/middleware.ts`) protege todas las rutas bajo `/admin/*`; sin sesión → redirige a login.
- Rutas exclusivas de admin (registros, ajustes, usuarios) bloqueadas a instructores; acceso ajeno → redirige a una ruta permitida.
- Toda acción sobre cursos y participantes valida `instructorId === session.user.id`; acceso ajeno → 403.
- Un usuario no puede borrar su propia cuenta (la action comprueba `targetUserId !== session.user.id`).
- El DNI se normaliza siempre antes de almacenarse o consultarse: `trim` + uppercase + strip de `\s\-_.` + regex `/^[A-Z0-9]{5,20}$/`. Aplica a todos los entry points (Zod schemas, server actions).
- En flujos de alta pública, el wizard verifica el DNI antes de avanzar al paso de pago: si ya tiene membresía vigente en la temporada activa, bloquea con mensaje + link a renovación.

### UC-AUTH-01 — Inicio de sesión y enrutado por rol · DONE

</summary>

- **Actor:** Usuario interno (Administrador o Instructor)
- **Trigger:** Accede a `/admin/login`

**Flujo principal**
1. Usuario introduce email y contraseña.
2. NextAuth verifica credenciales contra el hash. Inválidas → error, permanece en login.
3. Sistema crea sesión, redirige a `/admin` y aplica filtrado de rutas por rol; la sidebar muestra solo los apartados permitidos.
