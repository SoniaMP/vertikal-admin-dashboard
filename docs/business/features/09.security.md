# SEC — Seguridad de archivos de licencia

Endurecimiento del flujo de subida y descarga de PDFs de licencia federativa que adjuntan los inscritos a cursos. Hoy el endpoint de subida es público y la descarga es accesible con solo conocer el UUID del fichero — esta feature cierra ese gap y alinea el almacenamiento y el lifecycle con principios de RGPD.

## Reglas

- El PDF de licencia subido por un inscrito a curso es un dato personal sensible (DNI, fecha de nacimiento, foto): solo es accesible al admin o al instructor propietario del curso.
- Los inscritos nunca re-descargan su propio PDF: una vez subido, es estrictamente interno.
- Los archivos físicos viven fuera del repo (configurable vía `UPLOADS_DIR`).
- No hay audit log de descargas (fuera de alcance para una asociación pequeña).

## Casos de uso

(Los flujos de subida y descarga de PDFs viajan dentro de los UCs de inscripción a curso de [CRS](02.courses.md) y de gestión de participantes; SEC define las reglas y las pendientes técnicas.)

### Almacenamiento de ficheros fuera del repositorio

Hoy los PDFs viven en una ruta dentro del proyecto. Esto rompe en deploys (se pueden borrar al re-desplegar), interfiere con backups y mete datos personales en cualquier export del repo. Decisión: variable `UPLOADS_DIR` (default `data/uploads/` para local, `/var/lib/vertikal/uploads` en producción).

### Cascade delete del PDF al eliminar inscripción

Cuando se borra un `CourseRegistration` hoy, el PDF queda huérfano en disco. Decisión: la action de borrado elimina también el fichero. Idempotente: si el fichero ya no está, no error.

- [ ] La server action `deleteEnrollee` elimina el fichero de disco después de borrar la fila DB.
- [ ] Idempotencia: si el fichero ya no está, no lanza error.
- [ ] Tests: cascade delete con fichero presente, con fichero ausente.

### Limpieza nocturna de huérfanos y caducados

A pesar del cascade delete pueden quedar huérfanos (subidas que nunca completan inscripción) o ficheros caducados (cursos que terminaron hace tiempo). Decisión: script `npm run cleanup-uploads` con dos pasadas: (a) huérfanos = ficheros en `UPLOADS_DIR` con >24h de antigüedad sin `licenseFileKey` asociado en BD; (b) caducados = ficheros cuyo curso asociado terminó hace >6 meses.
