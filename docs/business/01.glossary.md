# Glosario

Términos del dominio utilizados en todo el código.

## Personas

| Término | Definición |
|---|---|
| **Socio** (Member) | Persona registrada en el club, identificada por DNI. Almacena datos de contacto y demográficos. |
| **Usuario** (User) | Cuenta de personal interno con acceso al panel de administración. Puede tener uno o más roles. |
| **Administrador** (Admin) | Usuario con el rol ADMIN. Acceso completo a todas las funcionalidades y datos. |
| **Instructor** (Instructor) | Usuario con el rol INSTRUCTOR. Gestiona sus propios cursos y participantes. |

## Membresías y licencias

| Término | Definición |
|---|---|
| **Temporada** (Season) | Periodo temporal acotado (p. ej. "2025-2026") que delimita precios, membresías y ofertas. |
| **Membresía** (Membership) | Inscripción por temporada que vincula a un Socio con una Temporada, junto con licencia, estado y datos de pago. |
| **Cuota de socio** (Membership Fee) | Importe **obligatorio** que paga toda Membresía. Configurado globalmente en Ajustes (`MEMBERSHIP_FEE`). Independiente de la federativa. |
| **Federado** (`isFederated`) | Marca **opcional** de una Membresía: indica que el socio se ha federado a través del club. La federación es opcional; un socio puede tener Membresía sin estar federado. |
| **Tipo de licencia** (LicenseType) | Categoría principal de licencia (p. ej. "Federada", "No Federada"). |
| **Subtipo de licencia** (LicenseSubtype) | Variante específica dentro de un Tipo de licencia. |
| **Categoría** (Category) | Clasificación secundaria de las licencias (p. ej. grupo de edad o tipo de actividad). |
| **Oferta de licencia** (LicenseOffering) | Punto de precio concreto para una combinación específica de Temporada + Tipo + Subtipo + Categoría. Solo aplica si la Membresía está federada. |

## Suplementos

| Término | Definición |
|---|---|
| **Suplemento** (Supplement) | Complemento opcional de pago que puede añadirse a una Membresía. |
| **Grupo de suplementos** (SupplementGroup) | Agrupación lógica de Suplementos con precio combinado opcional. |
| **Precio de suplemento** (SupplementPrice) | Precio por temporada de un Suplemento individual. |
| **Precio de grupo de suplementos** (SupplementGroupPrice) | Precio fijo combinado para cualquier combinación de Suplementos dentro de un grupo. |
| **Suplemento de membresía** (MembershipSupplement) | Tabla de unión que registra qué Suplementos incluye una Membresía, con captura de precio (snapshot). |

## Cursos

| Término | Definición |
|---|---|
| **Tipo de curso** (CourseType) | Categoría de cursos (p. ej. "Iniciación", "Perfeccionamiento"). |
| **Catálogo de cursos** (CourseCatalog) | Oferta concreta de un curso con fechas, capacidad, estado y, opcionalmente, instructor. |
| **Precio de curso** (CoursePrice) | Tramo de precio para un curso, con ventana de venta opcional. |
| **Inscripción a curso** (CourseRegistration) | Inscripción individual en un curso, con seguimiento de pago y confirmación. |

## Estados de curso

| Estado | Descripción |
|---|---|
| **DRAFT** | Creado por un instructor, todavía no visible en el sitio público. Pendiente de aprobación del administrador. |
| **ACTIVE** | Aprobado por el administrador. Visible en `/cursos`, acepta inscripciones y pagos. |
| **INACTIVE** | Desactivado por un administrador o por el instructor. Oculto del sitio público. |

## Autenticación y configuración

| Término | Definición |
|---|---|
| **Rol** (Role) | Nivel de permisos asignado a un Usuario (ADMIN, INSTRUCTOR). |
| **Rol de usuario** (UserRole) | Tabla de unión que mapea Usuarios con Roles. |
| **Configuración** (AppSetting) | Almacén clave-valor para la configuración en tiempo de ejecución (cuotas, emails de notificación, temporada activa, branding). |

## Pagos

| Término | Definición |
|---|---|
| **Stripe Checkout** | Flujo de pago externo. Las sesiones se crean mediante `getStripe()` en `src/lib/stripe.ts`. |
| **Webhook** | Stripe llama a `/api/webhooks/stripe` para confirmar el pago. Actualiza el estado de Membresía o Inscripción a curso. |
| **Snapshot** | Precio y etiqueta capturados en el momento de la creación en Membresía/Suplemento de membresía. Registro inmutable de lo cobrado. |
